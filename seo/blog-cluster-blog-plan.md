# MVP SEO Blog Plan

Aim: produce cluster-aligned blog posts that reinforce the Pompano Beach programmatic landing pages, capture long-tail informational intent, and funnel readers into the key Zillow-style slug groups. The MVP set focuses on 6 blog pages—one per high-value cluster—before scaling to 150 landing pages.

## Format
Each blog page follows this reusable SEO structure derived from `programmatic-page-template.md`:
1. H1 with primary keyword + city.
2. Intro paragraph summarizing why the topic matters (include cluster keyword, mention `Pompano Beach`).
3. Sectioned body (3–4 subsections) covering insights, data, and tips.
4. Inline call-to-action linking to the most relevant slug(s). Use descriptive anchors from `internal-linking-map.md`.
5. Internal CTA block referencing related clusters.
6. FAQ (3 questions) using keywords from `keyword-universe.md`.
7. Structured data: apply `BlogPosting` + `FAQPage` schema.

## Content components
- **Stats**: include market trends or IDX snapshots (avg price, inventory) for credibility.
- **Visual Hooks**: hero image + supporting callouts (e.g., neighborhood map, price tiers). Use `componentNotes` style guidance.
- **Internal linking**: embed 3–4 links referencing cluster-explained slugs using anchor text like "waterfront condos with a view".
- **CTA**: final block referencing a lead capture (tour request or market report download) tied to the slug focus.
- **Schema**: add `BlogPosting` with `mainEntityOfPage` pointing to the primary slug and `FAQPage` for the Q&A section.

## Launch rhythm
1. Draft one blog at a time with SEO metadata (title + description) derived from `keyword-universe.md` (match focus phrase). Validate uniqueness.
2. Publish to CMS, attach schema markup, and drop into the footer or sidebar navigation near the relevant cluster to funnel links.
3. After MVP set, rotate through remaining clusters using the same template.

This MVP blog set gives editors crawled content that supports the 150 landing pages and provides early internal links for topical authority. Use this plan to brief writers and confirm CMS fields align with the structured sections.
